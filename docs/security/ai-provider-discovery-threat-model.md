# AI Provider 模型发现威胁模型

日期：2026-07-22  
范围：Phase 5 / L5-002A

## 资产与信任边界

受保护资产包括 Provider 密钥、租户模型目录、内部网络、审计完整性和服务可用性。浏览器、用户输入的 Base URL、DNS 与第三方 Provider 均不受信任；只有 Logion API 的授权、加密、网络适配器和数据库边界受控。

## 威胁与控制

| 威胁                              | 控制                                                                                 | 验证                                  |
| --------------------------------- | ------------------------------------------------------------------------------------ | ------------------------------------- |
| SSRF、云元数据访问、DNS rebinding | HTTPS 与端口白名单；解析结果全量 `is_global`；固定已验证 IP；保留 Host/SNI           | 单元测试覆盖纯私网及公网+私网混合答案 |
| 环境代理绕过目标验证              | `trust_env=False`，不读取代理环境                                                    | 适配器构造约束与请求测试              |
| 重定向到内网                      | `follow_redirects=False`，3xx 稳定拒绝                                               | 302 归一化测试                        |
| 响应耗尽内存/数据库               | 流式读取；Content-Length 与实际字节双重上限；模型数和 ID 长度上限                    | 超限与恶意 ID 测试                    |
| 密钥泄漏                          | 服务端短时解密；响应、审计、模型表和错误均不含密钥；浏览器不持久化                   | 单元与集成断言                        |
| 跨租户或低权限探测                | `ai.configure`；外部工作区返回 404；Owner/Admin；recent-auth、Origin、CSRF、独立限流 | 集成角色矩阵                          |
| Provider 原始信息泄漏             | 不保存响应正文/头/异常；只返回稳定错误码                                             | 审计内容断言                          |
| AI 故障阻断学习                   | AI 请求独立失败并可重试；核心离线流程无依赖                                          | UI 与服务边界检查                     |

## 残余风险

公网 Provider 自身可记录请求来源和所用凭据；界面在发起检查前明确确认。公网 IP 在请求期间发生路由层劫持不由应用层解决，TLS 使用原域名 SNI 和证书验证降低该风险。模型能力字段在本包中保持保守默认值，后续不得依据未验证的 Provider 声明自动授予写入正式学习数据的能力。

## L5-002B 路由与预算增量

- 路由、模型覆盖和预算均按 workspace 查询并要求 `ai.configure`，跨租户 ID 返回 404。
- 客户端传入的模型 ID、顺序、能力、价格、币种、预算和版本均不可信；服务端重新验证归属、活动状态和乐观锁版本。
- 活动任务类型和路由名称由数据库部分唯一索引约束；路由目标同时约束位置与模型唯一。
- Token、上下文、价格和预算使用有界整数；成本由服务端按保存价格计算并向上取整。
- 预检只返回模型/Provider ID、候选顺序和估算数字，不返回密钥、Base URL、原始 Provider 数据或用户正文。
- 当前预检不预占月度额度；在 AIRun 原子预占落地前，它不能作为“已获执行额度”的凭证。
