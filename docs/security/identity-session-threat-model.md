# 身份与浏览器会话威胁模型

- 工作包：`L1-001`
- 状态：实现增量，合并前需独立安全复核
- 最后更新：2026-07-20
- 决策基线：`ADR-0002`

## 范围与资产

本增量覆盖邮箱密码注册/登录、浏览器 Cookie 会话、Refresh 轮换、CSRF、设备枚举与撤销、认证审计。Passkey、TOTP、恢复码和邮件验证属于后续 `L1-002`，不得把本增量视为完整账户恢复方案。

保护资产包括密码、Access/Refresh/CSRF Token、账户与设备标识、会话状态和审计记录。浏览器、API、PostgreSQL 与 Redis 之间均视为独立信任边界；生产流量必须经 HTTPS。

## 威胁与控制

| 威胁                     | 当前控制                                                                                | 验证证据                                                       |
| ------------------------ | --------------------------------------------------------------------------------------- | -------------------------------------------------------------- |
| 密码库泄漏与离线破解     | Argon2id；每条密码独立 salt；参数可检测并在登录时升级                                   | 单元测试验证哈希类型、正确和错误密码                           |
| 用户枚举、撞库与认证 DoS | 登录失败统一响应；不存在账户也执行 dummy hash；登录使用独立 IP 与账户桶，注册按 IP 限流 | 单元测试验证换邮箱不改变 IP 桶；限流服务不可用时默认拒绝       |
| Token 数据库泄漏         | Token 使用高熵随机值；数据库只保存基于服务端密钥的 HMAC-SHA256                          | 单元测试验证哈希稳定且随密钥变化                               |
| Refresh Token 窃取或重放 | Refresh 单次使用、事务行锁和逐次轮换；检测已使用 Token 时撤销整个会话                   | 集成测试重放旧 Token 并验证后续 Access 失效                    |
| CSRF                     | 写认证请求校验可信 Origin；双提交 CSRF Token 并与服务端会话哈希绑定                     | 集成测试在 Refresh 和设备撤销请求携带 CSRF；缺失或不匹配时拒绝 |
| Cookie 窃取              | Access/Refresh 为 HttpOnly；生产强制 Secure；SameSite 限制；错误和退出时清除            | 配置测试阻止不安全生产启动；集成测试验证清除                   |
| 丢失设备继续访问         | 用户可枚举设备并撤销；撤销设备同时撤销其全部服务端会话                                  | 集成测试覆盖旧设备与当前设备撤销                               |
| Refresh 并发竞争         | PostgreSQL `FOR UPDATE` 串行化同一 Token 的轮换                                         | Linux CI 使用真实 PostgreSQL 执行集成场景                      |
| 审计或日志泄密           | 审计仅保存邮箱、IP、User-Agent 的 HMAC 摘要；校验错误不回显请求输入                     | 单元测试验证密码不出现在 422 响应；人工复核日志配置            |
| 配置错误                 | 生产拒绝开发密钥、短密钥、非 Secure Cookie、关闭 Origin 校验和 HTTP Origin              | 配置单元测试与 Compose 配置解析                                |

## 剩余风险与后续工作

- 已完全控制的终端、恶意扩展或同站点 XSS 仍可读取页面内容和可读 CSRF Cookie；前端必须继续执行 CSP、依赖和 Markdown XSS 控制。
- Refresh 严格重放检测可能把同一浏览器的异常并发刷新识别为重放并要求重新登录。客户端在后续接入时必须对单设备刷新请求做 single-flight；是否增加短暂 grace window 需独立 ADR 和安全复核。
- 设备撤销不能抹除已离线保存的用户内容。Phase 2 必须实现本地锁、撤销后停止同步及明确风险提示。
- Redis 当前只承载限流而非权威认证状态；Redis 不可用时注册和登录失败关闭，已有会话认证仍由 PostgreSQL 决定。
- `L1-002D` 新增邮箱优先、统一 `202` 的注册流程，验证前不设置密码或创建会话。旧 `/register`
  仅为开发和测试兼容保留，生产配置禁止启用；支持客户端迁移完成前不得删除旧 OpenAPI。
- Passkey 与 TOTP/恢复码已经实现；邮件验证核心见
  `email-verification-account-recovery-threat-model.md`。真实邮件投递、密码找回和安全通知仍待后续
  工作包交付。

## 合并与运行要求

- 身份代码和迁移必须由非实现者独立复核；仓库只有一名人类维护者时不得降低此要求。
- CI 必须完成迁移 `upgrade -> downgrade -> upgrade`、真实 PostgreSQL/Redis 集成测试、OpenAPI 兼容检查和依赖/密钥扫描。
- 生产 Secret 轮换会使现有 Token HMAC 无法匹配，相当于全局退出。轮换前应告知用户并监控登录失败率。
- 数据库回滚会删除本工作包创建的身份、设备、会话和审计表，只允许在确认无生产身份数据时执行；有数据后采用前向修复。
