# 搜索、通知与日历威胁模型

日期：2026-07-22  
范围：Phase 5 / L5-012

| 威胁                                  | 控制                                                                                                            | 验证                                         |
| ------------------------------------- | --------------------------------------------------------------------------------------------------------------- | -------------------------------------------- |
| 通过搜索猜测其他租户或私有 Space 内容 | 先校验 workspace membership，再以授权 Space ID 约束每类查询；私有 Space 仅 owner 可见；个人论文额外限制 user ID | 跨用户 private Space 与跨 workspace 集成负测 |
| `%`、`_` 或反斜线扩大匹配范围         | Pydantic 限制查询长度；ORM 参数化；显式转义 LIKE 元字符                                                         | 通配符按字面搜索测试                         |
| 查询正文进入 URL、日志或第三方服务    | POST request body；不生成含查询的审计事件；不接入第三方托管搜索                                                 | OpenAPI 与日志字段审查                       |
| 搜索片段过长导致正文外泄              | 结果仅返回有界片段，类型、Space 和权限来源显式展示                                                              | 响应 schema 与长度测试                       |
| 离线搜索绕过本地加密                  | protected payload 只能通过已解锁 `OfflineVault` 解密；锁定后失败关闭                                            | IndexedDB 单元测试与密文静态断言             |
| 权限撤销后离线缓存短暂可读            | 沿用同步 tombstone、epoch 和 rebootstrap；产品明确离线撤销延迟                                                  | 同步恢复测试；不宣称即时远程擦除             |
| 通知成为敏感正文副本                  | title 和 summary 设置数据库长度上限；生产端口截断；不存附件、Token 或正文                                       | 模型约束、集成测试与代码审查                 |
| 用户关闭关键安全通知                  | schema 强制 enabled categories 包含 `security`；生产端口忽略关闭安全类别的偏好                                  | 偏好校验与 emit 测试                         |
| 通知目标 ID 被当作授权                | 通知列表按 recipient 隔离；后续目标读取仍执行原对象授权                                                         | IDOR 负测和目标 API 权限测试                 |
| 日历 Token 被数据库读取或日志泄漏     | 256-bit URL-safe Token 只返回一次；数据库只存用途域 HMAC；审计不含 Token                                        | 数据库和审计明文断言                         |
| 枚举或暴力读取日历订阅                | Token 字符白名单和长度限制；按隐私哈希 IP 限流；不存在、撤销和无 membership 统一 404                            | 公共端点与撤销测试                           |
| 日历泄漏私人正文                      | 只投影任务/考试/复习标题和时间；不读取笔记、附件、错题、任务描述                                                | ICS 内容精确断言                             |
| ICS 注入和响应缓存泄漏                | 转义反斜线、逗号、分号和换行；UTF-8 折行；`private, no-store` 与 `no-referrer`                                  | 特殊字符单元测试和响应头断言                 |
| 成员退出后订阅继续工作                | 每次 feed 读取重新校验 active membership；用户可立即 revoke                                                     | membership/revoke 负测                       |

残余风险：第三方日历客户端可能同步或转发已读取的事件，Logion 无法远程删除客户端已经复制的数据。产品必须把订阅 URL 标示为秘密，并建议泄漏后撤销重建；首版不向日历投影正文或附件，以降低该风险。
