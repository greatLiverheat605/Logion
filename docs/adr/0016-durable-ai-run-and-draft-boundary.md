# ADR 0016: AI 执行采用持久运行、预算预占和草稿隔离

- 状态：Accepted
- 日期：2026-07-22
- 范围：Phase 5 / L5-003

## 决策

云 AI 请求先创建持久 `AIRun`，不在浏览器请求中直接完成。创建时要求用户明确确认发送范围，服务端重新解析路由并在同一 PostgreSQL 事务中预占当月 Token 与成本；相同用户、工作区和幂等键只创建一次运行。输入字段使用与 Provider 凭据分域的 AAD 信封加密，默认在终态立即清除。

Worker 使用 `FOR UPDATE SKIP LOCKED` 领取运行。仅网络/超时、429 和特定 5xx 归一化错误允许有限重试与按配置顺序降级；认证、预算、结构化 schema、状态错误和取消不重试。取消 queued 运行立即释放预算，running 运行记录取消请求并在请求流、重试和提交草稿前检查。

Provider 输出必须是与请求字段完全一致的字符串 JSON 对象，并通过响应大小、字段、Token 和领域边界校验后写入 `AIOutputDraft`。批准、编辑或拒绝只改变草稿决策，本工作包明确不修改正式目标对象；后续领域专用 apply 适配器必须重新验证目标权限和当前版本。

## 理由

同步 HTTP 生成无法可靠支持断线、刷新、重试、取消和预算并发。仅在完成后记账会造成并发超支。让 AI 直接写正式对象会绕过人工授权、版本冲突和领域不变量。

## 后果

- Worker 暂停不影响学习核心；运行保留 queued 状态并可取消。
- 实际用量结算从预留转入已消费；失败和取消释放预留。
- Provider 不支持幂等时，进程在外部请求完成但本地提交前崩溃仍可能产生不可自动归因的外部费用；当前不自动重放 running 状态，需运维审查。
- `accepted` 表示草稿经人工审查，不表示正式对象已经更新；界面和 API 不得暗示自动应用。
