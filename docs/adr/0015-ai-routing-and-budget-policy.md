# ADR 0015: AI 路由与预算由工作区配置并在服务端解析

- 状态：Accepted
- 日期：2026-07-22
- 范围：Phase 5 / L5-002B

## 决策

AI 任务类型、模型能力覆盖、主备顺序和预算均是工作区配置，不写死任何课程、专业、考试或研究方向。一个启用的任务类型只能对应一个活动路由；路由保存最多十个有序模型目标。解析时按顺序检查 Provider/模型启用状态、最近健康状态、结构化与流式能力、上下文窗口、币种和预算，返回主选及有限降级候选。

模型价格以“每百万 Token 的最小货币单位”保存，估算结果向上取整。工作区可分别限制月度 Token 与成本；本包的路由预检不预占额度，实际运行的原子预占、结算和取消释放由 AIRun 工作包实现。

## 理由

前端路由会被绕过，也无法可靠处理并发预算。Provider 内部模型名称和能力声明不可信，因此探测值与管理员覆盖值必须分离。显式有序路由比隐式“智能选择”更可解释、可测试，也能在单一 Provider 故障时安全降级。

## 后果

- 路由、预算和模型覆盖仅允许 `ai.configure`，写入要求 recent-auth、Origin、CSRF、限流和审计。
- 不健康、停用、能力不足、上下文不足或币种不匹配的模型不会成为候选。
- 认证失败、预算超限、无有效路由和用户取消不属于可盲目重试的错误。
- L5-002C/L5-003 必须使用同一解析服务与数据库预算权威值，不能信任客户端候选或估算金额。
