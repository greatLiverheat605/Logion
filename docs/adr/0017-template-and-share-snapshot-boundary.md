# ADR 0017: 模板使用结构化副本，分享使用不可变最小快照

- 状态：Accepted
- 日期：2026-07-22
- 范围：Phase 5 / L5-010、L5-011

## 决策

首个模板类型是通用 `goal_plan`，只能从当前用户已授权的结构化目标、计划版本和阶段生成。模板包保存 schema/product 版本、作者、许可证、locale、目标人群、变更日志、内容哈希、风险元数据和去身份对象图。每个版本不可变；安装时重新执行 schema 与领域校验，并为目标、计划、版本和阶段生成全新 ID，在一个事务中创建独立副本。

私有 Space 只能生成仅创建者可见的私有模板；workspace 可见模板必须来自共享 Space。模板升级只增加新版本，不修改已安装对象。

只读分享不暴露活动对象或 ACL。用户从白名单中勾选字段，服务端在创建时生成不可变投影。高熵 Token 仅首次返回，数据库只保存带用途域的 HMAC；默认 30 天、最长 365 天，可即时撤销。公开读取仅返回快照、标题和到期时间。

## 理由

直接引用活动对象会让后续编辑、权限变化或间接链接扩大公开范围。导入任意 JSON 或脚本会引入越权、恶意链接、ID 冲突和执行风险。不可变包和快照能提供稳定审计、明确撤销语义和可重复安装。

## 后果

- 当前模板仅覆盖目标—计划—阶段结构；后续对象类型必须增加版本化 schema 和安装适配器。
- 公共模板审核/市场仍不开放；workspace 模板不是公开发布。
- 分享撤销或过期后统一返回 404；Token 不进入列表、审计或数据库明文。
- 已安装模板不因源模板撤回或升级而删除或覆盖，只能收到后续差异建议。
