# ADR 0014: Provider 模型发现采用解析后固定公网地址

- 状态：Accepted
- 日期：2026-07-22
- 范围：Phase 5 / L5-002A

## 决策

OpenAI-compatible Provider 的模型发现只允许已保存且通过静态校验的 HTTPS Base URL。每次请求都重新解析域名；解析结果非空且**所有地址**均为全局公网地址时才放行。连接目标固定为本次已验证 IP，HTTP `Host` 与 TLS SNI 保留原始域名。客户端禁用环境代理和重定向，端口、连接/总超时、响应体、模型数量与模型 ID 均有上限。

Provider 凭据仅在服务端解密并用于该次请求。系统不保存原始响应、响应头或异常正文，只保存模型最小元数据、健康状态、检查时间和稳定错误码。

## 理由

仅在保存配置时检查 URL 无法阻止 DNS rebinding；仅检查一个解析地址也会遗漏混合公网/私网答案。解析后固定地址可避免连接阶段再次解析到内网，同时保留域名验证所需的 Host/SNI。

## 后果

- DNS 返回任一私网、回环、链路本地、保留或其他非全局地址时整体拒绝。
- 3xx 不跟随；Provider 代理跳转必须在配置中显式使用最终公开地址。
- 健康检查失败只影响 AI 能力，不影响离线学习、任务、笔记和研究核心流程。
- 后续生成调用必须复用同一网络边界，不能自行创建宽松 HTTP 客户端。
