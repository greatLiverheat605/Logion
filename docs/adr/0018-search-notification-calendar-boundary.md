# ADR 0018: 搜索、通知与日历采用权限优先和最小披露边界

- 状态：Accepted
- 日期：2026-07-22
- 范围：Phase 5 / L5-012

## 决策

统一搜索在查询任何业务表之前解析 workspace 成员关系与 Space 可见范围。在线查询通过带 CSRF 校验的 POST 正文提交，不进入 URL；服务端不记录完整查询正文。首版只搜索目标、任务、笔记、资料和当前用户的论文记录，并返回对象类型、workspace、Space、有限片段和权限来源。离线搜索只处理当前设备已经同步、尚未删除且能够由已解锁保险箱读取的实体，不向第三方搜索服务发送正文。

产品内通知按用户和 workspace 隔离。通知记录只保存标题、最多 500 字摘要、可选目标引用和去重键，不复制对象正文。用户可以关闭非安全类别，但 `security` 类别始终启用；通知引用不是访问凭证，目标对象仍需在访问时重新授权。

iCalendar 订阅使用高熵 bearer Token。Token 只在创建响应中返回一次，数据库只保存带用途域的 HMAC。每次读取都校验 feed 状态和用户的有效 workspace membership。订阅只投影有截止时间的任务、考试和复习主题，不包含任务描述、笔记、附件、错题或研究正文；响应使用 `private, no-store`，订阅可立即撤销。

## 理由

先搜索后过滤会产生跨 Space 片段泄漏，URL 查询会进入浏览历史、代理和分析日志。通知若复制正文，会形成难以随源对象权限和删除同步的第二份敏感数据。日历客户端通常把订阅 URL 当作长期 bearer secret，因此必须限制投影内容、避免明文落库并提供撤销路径。

## 后果

- 首版在线搜索使用 PostgreSQL 参数化 `ILIKE`，不引入第三方全文索引；规模门槛达到后可替换实现，但权限解析和结果契约不变。
- 离线权限撤销存在同步延迟；设备再次联网后必须按既有同步协议接收 tombstone 或重新 bootstrap。
- 安静时间和时区偏好在本阶段持久化；邮件、Web Push 和定时调度属于后续独立工作包，不能用本地通知伪装云端已完成。
- 日历订阅是只读快照视图，不提供第三方日历双向写入；Token 泄漏时的补救方式是撤销并创建新 feed。
