## 1. 针对你当前情况的判断

你的优势是计算机专业背景、每天时间充足、英文论文阅读没有明显障碍，而且已经有 Vigils 这个真实项目作为实验环境。

当前主要短板不是“Agent 知识不够新”，而是科研地基偏弱：

- Python 尚不足以流畅实现和修改模型；
- PyTorch 尚未入门；
- 线性代数只能支持非常初步的理解；
- 概率统计尚未建立体系；
- 如果立即阅读复杂的序列推荐、图对比学习或元优化论文，很容易停留在“看懂摘要和结构图”，无法理解公式、复现实验或判断方法是否合理。

因此，47 天的优先级应为：

1. Python 与科研编程；
2. 线性代数、概率、微积分和优化的最低必要基础；
3. PyTorch 与机器学习实验；
4. 推荐系统、Transformer、对比学习；
5. 导师论文复现与研究选题；
6. Agent 安全作为研究问题和数据来源，而不是同时铺开所有安全工程知识。

### 开学前不应追求的目标

- 完整学完 CS229、CS224W 或一本深度学习教材；
- 从零复现导师最复杂的元优化论文；
- 同时掌握 LangGraph、LlamaIndex、smolagents 等多个框架；
- 从零训练大语言模型；
- 在数学基础不足时大量阅读论文但不做推导和代码；
- 为了“每天学满”而牺牲睡眠、运动和复盘。

## 3. 每天 6 小时的固定结构

除阶段验收日外，每天按以下结构执行：

| 时间 | 内容 | 要求 |
|---|---|---|
| 2 小时 | 主课程/教材 | 看完后必须关闭资料，用自己的话写摘要 |
| 2 小时 | 编程练习/模型实验 | 不复制整段答案；每次提交可运行代码 |
| 1 小时 | 数学推导/习题 | 至少手写或在 Markdown 中完成 3–5 题 |
| 1 小时 | 论文、Vigils 或当日复盘 | 每天必须留下可检查的文字产出 |

建议每学习 50 分钟休息 10 分钟。每天 6 小时指高质量学习时间，不包括吃饭和长时间刷视频。

### 每日结束前回答四个问题

1. 今天真正理解了什么？
2. 哪个知识点还只能复述、不能解释？
3. 今天的代码能否从干净环境重新运行？
4. 明天最重要的唯一任务是什么？

### 每周节奏

- 周一至周五：新知识 + 实验；
- 周六：综合项目和论文；
- 周日：前 3 小时测验与查漏，后 3 小时轻量复盘、整理和休息；
- 如果疲劳明显，周日只保留 3 小时。少学半天比连续低效更划算。

- 输出风险矩阵。

### 8 月 27 日：轨迹数据设计

- 定义 trace、step、tool、resource、subject、decision、effect、risk、label；
- 明确原始敏感参数不能进入训练数据；
- 定义正常、误操作、恶意、未知四类标签；
- 设计训练/验证/测试按场景或时间切分；
- 写数据字典和 10 条示例。

### 8 月 28 日：攻击与正常样本

- 建立至少 15 条正常任务；
- 建立至少 15 条攻击/高风险任务；
- 覆盖敏感文件、出站网络、命令执行、密钥、工具描述投毒；
- 每条样本记录攻击者能力和预期安全结果；
- 不执行真实破坏，只使用隔离测试资源。

### 8 月 29 日：风险检测基线

- 基线 1：规则分数；
- 基线 2：TF-IDF/统计特征 + 逻辑回归；
- 基线 3：工具 ID Embedding + MLP/平均池化；
- 报告 Precision、Recall、FPR、FNR 和混淆矩阵；
- 分析最危险的漏报。

### 8 月 30 日：第六周验收

- 完成至少 30 条正常和 30 条攻击/高风险轨迹；
- 完成三种基线或至少前两种；
- 完成 Vigils 威胁模型 v1；
- 写研究问题、假设和评测指标；
- 明确当前数据集的偏差与不可发表之处。

**本周门槛**：研究问题必须能够被实验否定。例如：“加入结构与语义双视图对比学习，是否能提高未见攻击场景的 PR-AUC，同时将正常任务 FPR 控制在某阈值以下？”

## 11. 建议使用的主课程

资源越多越容易分散。47 天内只设置以下主线：

### Python

- 主线：Python 官方教程或一门系统基础课程；
- 练习：每天 8–15 道基础题；
- 原则：视频/阅读不超过当天 Python 时间的一半。

### 数学

- 线代直觉：3Blue1Brown《线性代数的本质》；
- 概率：选择一门面向计算机/机器学习的基础概率课；
- 查漏：StatQuest；
- 只学路线中明确需要的章节，不通刷高等数学课程。

### 深度学习

- 主教材：[Dive into Deep Learning](https://d2l.ai/)；
- 重点：预备知识、线性网络、MLP、优化、注意力与 Transformer；
- 不要求 47 天内完整学习 CNN、检测、分割等章节。

### 推荐系统

- 实验框架：[RecBole](https://recbole.io/)；
- 基础论文：BPR、SASRec、LightGCN；
- 导师论文：先精读 TKDE 2024 的可学习模型增强对比学习论文。

### Agent 与安全

- Agent 入门：[Hugging Face Agents Course](https://huggingface.co/learn/agents-course/) Unit 1；
- 结构化补充：[DeepLearning.AI Agentic AI](https://www.deeplearning.ai/courses/agentic-ai)；
- 安全必读：[OWASP LLM Top 10](https://genai.owasp.org/llm-top-10/)；
- Agent 专项：[OWASP Agentic AI Threats and Mitigations](https://genai.owasp.org/resource/agentic-ai-threats-and-mitigations/)；
- 攻击知识库：[MITRE ATLAS](https://atlas.mitre.org/)。

注意：Hugging Face Agents Course 在开学前只要求完成基础单元。不要为了证书挤占 PyTorch、推荐系统和论文时间。

## 12. 每周评分表

每周日按 0–2 分评分，总分 20 分：

| 项目 | 0 分 | 1 分 | 2 分 |
|---|---|---|---|
| 出勤 | 少于 4 天 | 4–5 天 | 6–7 天 |
| 代码 | 大量不能运行 | 基本可运行 | 可复现、有说明 |
| 数学 | 只看不做 | 能做基础题 | 能推导并解释 |
| 笔记 | 无 | 摘抄为主 | 自己组织、有疑问 |
| 测验 | <60% | 60%–79% | ≥80% |
| 实验 | 无结果 | 有结果无分析 | 有对照和错误分析 |
| 论文 | 未读 | 能总结 | 能质疑假设和实验 |
| Vigils | 无产出 | 有设计 | 有测试/数据证据 |
| 复盘 | 无 | 罗列任务 | 能调整下一周 |
| 身体状态 | 明显透支 | 一般 | 睡眠运动正常 |

- 16–20 分：按计划推进；
- 12–15 分：下一周减少 20% 新内容，增加复习；
- 低于 12 分：停止增加课程，回退到最近未通过的阶段门槛。

## 13. 给导师汇报时的建议表达

不要把自己包装成已经掌握 Agent 安全或推荐系统。更可信的表达是：

> 我目前在补机器学习、PyTorch 和推荐系统基础，并用 RecBole 复现 BPR、LightGCN/SASRec。我参与 Vigils 的架构规划和 AI 开发，因此对 Agent 工具调用的安全问题比较关注。我初步想把 Agent 工具调用建模为行为序列，研究序列建模和对比学习能否提升未见攻击的检测能力，同时控制正常任务误报。这个方向与您的序列推荐、图学习和对比学习研究可能存在结合点。我会先完成严格基线和数据定义，希望听取您对研究问题、数据可信度和创新性的意见。

这种表达体现了准备、诚实和研究意识，也给导师留下调整方向的空间。

## 14. 开学后第一个月的默认计划

如果导师没有立即指定新任务：

- 第 1 周：根据导师意见修改研究问题，补概率与 PyTorch；

> 学基础 → 写代码 → 做实验 → 读论文 → 提出可证伪问题 → 汇报 → 根据证据修改。
