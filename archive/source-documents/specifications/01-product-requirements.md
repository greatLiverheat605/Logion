# 研途 Lab 产品需求规格说明书（PRD）

> 文档版本：1.0  
> 产品形态：个人、云端同步、离线优先的长期学习与科研管理系统  
> 本文不规定视觉样式，只规定行为、数据和验收标准。

---

## 1. 产品目标

研途 Lab 用于长期组织个人学习和科研活动。系统必须把“计划、执行、证据、掌握、复习、审查、论文与实验”连接成闭环，同时允许所有核心学习内容在离线状态下完整编辑。

核心目标：

- 打开系统后能确定当前最重要的学习任务；
- 每项任务都能关联知识点、资料、练习与验收标准；
- 学习活动产生可追溯记录和证据；
- 通过复习与审查发现遗忘、拖延和虚假完成；
- 论文、实验和项目目标能够与日常学习关联；
- 所有长期数据可同步、备份、恢复和导出；
- AI 可替换、可配置、可审计，不成为核心依赖。

## 2. 用户与使用场景

### 2.1 用户

第一版只有一个账户所有者。用户可以在多台可信设备上登录和使用，不存在普通用户、教师或管理员之间的业务权限差异。

### 2.2 主要设备

- Windows 电脑：计划、笔记、论文与实验的主要编辑设备；
- 手机：查看今日任务、计时、快速笔记、复习和上传截图；
- 未来平板：阅读与批注，可复用 PWA。

### 2.3 核心旅程

#### 旅程 A：每日学习

1. 查看今日计划、到期复习和阻塞项；
2. 选择任务并开始学习会话；
3. 打开关联资料；
4. 记录理解、问题和错误；
5. 提交证据；
6. 执行验收；
7. 安排复习；
8. 完成每日审查。

#### 旅程 B：断网学习

1. 用户打开已安装 PWA；
2. 本地解锁工作副本；
3. 查看和修改所有已同步任务、资料和笔记；
4. 完成计时、复习和审查；
5. 添加附件进入待上传状态；
6. 联网后自动或手动同步；
7. 用户处理少量不能自动合并的冲突。

#### 旅程 C：论文与实验

1. 创建论文卡片；
2. 关联知识点、研究问题和资料；
3. 记录公式、方法、实验和局限；
4. 创建可证伪研究假设；
5. 登记实验运行及其代码、数据和指标；
6. 记录失败和下一实验；
7. 在周审查中评估研究推进情况。

#### 旅程 D：AI 辅助

1. 用户配置一个 OpenAI-compatible Provider；
2. 测试连接并发现模型；
3. 为“周报”“论文卡片”等任务指定模型；
4. 用户主动选择内容并运行 AI；
5. 系统保存请求元数据和草稿输出；
6. 用户接受、编辑或拒绝；
7. 正式内容保留 AI 来源和确认记录。

## 4. 功能需求

### 4.1 账户、登录与设备

#### FR-AUTH-001 单用户初始化

- 系统首次启动时允许创建唯一所有者账户；
- 初始化完成后关闭公开注册；
- 重复初始化请求必须拒绝。

#### FR-AUTH-002 Passkey 登录

- 用户可注册多个 Passkey；
- Passkey 是推荐的日常登录方式；
- 新增或删除 Passkey 需要最近一次强认证；
- 不允许删除最后一种有效恢复方式。

#### FR-AUTH-003 密码与 TOTP 恢复

- 用户设置恢复密码；
- 恢复密码使用 Argon2id；
- 用户绑定 TOTP；
- 密码登录必须同时验证 TOTP；
- TOTP secret 加密保存。

#### FR-AUTH-004 恢复码

- 系统生成一次性恢复码；
- 服务端只保存哈希；
- 使用后立即失效；
- 重新生成后旧恢复码全部失效。

#### FR-AUTH-005 设备管理

- 展示设备名、类型、注册时间、最后同步时间和最后 IP 摘要；
- 允许撤销指定设备；
- 被撤销设备不能继续刷新 Token 或同步；
- 下次联网时清除其服务器同步权限。

#### FR-AUTH-006 离线解锁

- 至少成功在线登录一次后才能启用离线解锁；
- 离线解锁只能访问当前设备缓存；
- 离线状态禁止修改账户和安全配置；
- 用户可配置自动锁定时间；
- 多次失败触发等待时间。

### 4.2 学习计划与版本

#### FR-PLAN-001 创建计划

- 支持创建短期、学期、年度和长期计划；
- 计划具有日期、目标、状态和每日预算；
- 支持从 Markdown 导入。

#### FR-PLAN-002 计划版本

- 导入、批量重排或重大修改可生成计划版本；
- 版本保存创建时间、来源、说明和父版本；
- 历史版本只读；
- 可以比较版本差异；
- 恢复旧版本时创建新版本，不删除后续历史。

#### FR-PLAN-003 阶段与门槛

- 计划包含阶段；
- 阶段可设置开始/结束日期和验收门槛；
- 未通过门槛时系统提示补救，不强制锁死后续内容；
- 用户越过门槛时必须记录理由。

#### FR-PLAN-004 重排

- 支持延期、提前、拆分、合并、跳过和取消任务；
- 修改必须记录原因；
- 自动建议不得直接修改正式计划；
- 同一任务延期达到阈值后产生审查发现。

### 4.3 知识点

#### FR-TOPIC-001 知识树

- 知识点支持父子层级；
- 支持多前置依赖；
- 禁止产生依赖环；
- 支持标签、优先级和领域。

#### FR-TOPIC-002 掌握状态

- 状态为未接触、已浏览、能解释、能应用、能迁移；
- 升级掌握度应关联证据；
- AI 只能建议，不能直接确认；
- 保留掌握度历史。

### 4.4 任务

#### FR-TASK-001 任务内容

任务至少支持：

- 标题和说明；
- 日期和预计时长；
- 类型和优先级；
- 关联知识点；
- 前置任务；
- 主资料、补救资料和练习；
- 验收标准；
- 证据要求；
- 复习规则。

#### FR-TASK-002 状态机

允许的主要状态：草稿、已安排、进行中、已浏览、已练习、已提交、已验收、阻塞、延期、跳过和取消。

- 未提交证据不得进入“已验收”，除非验收标准明确无需证据；
- 状态变化形成追加事件；
- 历史状态不可静默覆盖。

#### FR-TASK-003 今日任务

今日任务聚合：

- 当日原计划；
- 到期复习；
- 延期任务；
- 补救任务；
- 临时新增任务；
- 已确认的 AI 建议。

### 4.5 学习会话

#### FR-SESSION-001 计时

- 支持开始、暂停、继续和结束；
- 支持手动补录；
- 服务端保存事件而非仅总时长；
- 检测跨设备重叠会话；
- 离线计时不依赖服务器时间。

#### FR-SESSION-002 学习总结

会话可记录：理解、疑问、错误、下一步、专注度、难度、学习前后信心和使用资料。

### 4.6 笔记

#### FR-NOTE-001 Markdown 笔记

- Markdown 编辑与预览；
- 自动保存；
- 关联多个业务对象；
- 支持标签和固定链接；
- 支持全文导出。

#### FR-NOTE-002 离线并发

- 笔记正文采用 CRDT 更新；
- 多设备离线修改正文可以合并；
- 标题等元数据冲突时保留双方选择；
- 定期生成可读 Markdown 快照；
- 保留版本历史。

### 4.7 资料库与 PDF 索引

#### FR-RESOURCE-001 资源管理

- 支持链接、课程、视频、书籍、论文、官方文档、练习和代码仓库；
- 保存难度、语言、预计时长、版本、标签和使用状态；
- 资源可关联任务与知识点；
- 链接检查必须防止 SSRF。

#### FR-RESOURCE-002 PDF 索引

- 保存 PDF 元数据、URL、本地路径引用、校验值、页数和笔记；
- 默认不上传 PDF 正文；
- 其他设备无法访问本地路径时显示不可用状态；
- 用户可以在不同设备重新定位同一校验值文件。

### 4.8 附件与证据

#### FR-ATTACH-001 附件

- 支持少量截图、图表、CSV、JSON、日志和小型文件；
- 默认单文件上限 20 MB，最终由配置决定；
- 离线保存本地 Blob；
- 联网后上传、校验和绑定；
- 上传使用幂等操作；
- 服务端不信任客户端 MIME。

#### FR-EVIDENCE-001 证据

- 证据可来自笔记、附件、URL、测验、实验或文字说明；
- 证据可以被多个验收引用；
- 删除证据需要检查依赖；
- 删除和替换产生审计事件。

#### FR-VERIFY-001 验收

- 验收包含标准、证据、规则结果、自评、AI 建议和最终确认；
- 失败时记录原因并可创建补救任务；
- AI 结果始终非最终决定。

### 4.9 复习、测验与错题

#### FR-REVIEW-001 复习调度

- 新知识可生成 D+1、D+3、D+7、D+14 复习；
- 支持简化 SM-2；
- 错误后缩短间隔；
- 重要知识可永久保留复习；
- 调度算法版本化。

#### FR-QUIZ-001 测验

- 支持选择、简答、代码、计算和自评题；
- 保存每次尝试；
- 记录正确性、耗时、信心和错误类型；
- AI 生成题必须标记来源并经用户确认后进入正式题库。

#### FR-ERROR-001 错误模式

- 可把多道错题归入同一错误模式；
- 错误复发触发审查；
- 错题不可因后来答对而删除历史。

### 4.10 审查

#### FR-AUDIT-001 每日审查

至少包含：计划/实际时间、证据完整性、疑问、错题、待同步内容和次日调整。

#### FR-AUDIT-002 每周审查

至少包含：时间偏差、验收任务比例、复习完成率、错题复发、延期、代码/论文/实验产出和下周行动。

#### FR-AUDIT-003 自动发现

确定性规则可产生发现，例如：

- 时长明显偏离；
- 只有观看没有练习；
- 无证据完成；
- 阻塞超过两天；
- 延期达到三次；
- 实验缺少版本信息。

规则必须显示依据和版本。

### 4.11 科研

#### FR-PAPER-001 论文卡片

- 保存论文元数据、引用、原始链接、代码链接和阅读状态；
- 记录问题、假设、方法、公式、实验、结论和局限；
- 支持 BibTeX 导入导出；
- 支持论文之间的引用、对比和扩展关系。

#### FR-RQ-001 研究问题

- 研究问题版本化；
- 保存支持和反对证据；
- 关联论文、实验和反馈；
- 修改必须记录原因。

#### FR-EXP-001 实验

- 保存假设、数据版本、代码版本、环境、配置、随机种子、指标和结论；
- 一个实验可以包含多次运行；
- 失败运行保留；
- 未提供代码和数据版本时不能标记为可复现。

### 4.12 AI 配置与使用

#### FR-AI-001 Provider

- 创建、编辑、停用和测试 Provider；
- 支持自定义 Base URL 和 API Key；
- 第一适配器为 OpenAI-compatible；
- 密钥仅服务端加密保存；
- Provider 安全修改需要在线和最近认证。

#### FR-AI-002 模型

- 从 Provider 探测模型；
- 允许手动添加；
- 保存上下文、结构化输出、工具、视觉和价格能力；
- 支持启停模型。

#### FR-AI-003 路由

- 每类 AI 任务有首选和备用模型；
- 可设置预算、超时和敏感内容策略；
- 失败时按配置降级；
- 路由决策可追溯。

#### FR-AI-004 草稿与确认

- AI 输出保存为草稿；
- 用户可以接受、编辑或拒绝；
- 接受后保存来源、模型和草稿版本；
- AI 不得自动修改掌握度、验收和正式计划。

### 4.13 同步与离线

#### FR-SYNC-001 本地优先写入

- 所有核心写操作先提交 IndexedDB；
- 本地成功后立即反馈；
- 网络失败不回滚本地操作；
- 每次变更生成 Outbox 操作。

#### FR-SYNC-002 增量同步

- 客户端批量推送操作；
- 服务端返回逐项结果；
- 客户端按 cursor 拉取变更；
- 操作具备幂等键；
- 删除使用 tombstone。

#### FR-SYNC-003 冲突

- 笔记正文使用 CRDT；
- 结构化实体使用 version；
- 不安全的冲突不能自动合并；
- 用户可查看并解决冲突；
- 冲突解决形成审计事件。

### 4.14 导入、导出与备份

#### FR-IO-001 导入

- 导入现有 Markdown 学习计划；
- JSON 导入使用带版本 schema；
- 导入先预览，确认后写入；
- 导入失败不得留下半成品。

#### FR-IO-002 导出

- Markdown、JSON、CSV 和 BibTeX；
- 不导出明文密码、Token、TOTP secret 和 AI Key；
- 导出记录审计事件。

#### FR-BACKUP-001 备份恢复

- 服务器自动备份 PostgreSQL、附件和必要配置；
- 备份加密并带校验值；
- 显示最后成功时间；
- 恢复前创建当前快照；
- 支持恢复演练。

## 5. 非功能需求

### NFR-001 可用性

- 核心学习流程在 AI 不可用时正常工作；
- 已安装 PWA 在断网时可启动；
- 未同步数据状态始终可见。

### NFR-002 性能

- 常用本地读取在中等设备上应近似即时；
- 在线普通 API 的服务端 p95 目标低于 500 ms，不含 AI；
- 首次同步支持分页和断点；
- 10 年个人学习规模下核心查询无需归档才能工作。

容量设计基线：

- 100,000 个任务；
- 1,000,000 个学习/状态/审计事件；
- 50,000 篇笔记或资源；
- 10,000 个附件；
- 5,000 篇论文；
- 100,000 次 AI 调用记录。

### NFR-003 安全

- 公网 HTTPS；
- 密码 Argon2id；
- Passkey/TOTP/恢复码；
- API Key 与 TOTP secret 加密；
- Markdown 安全渲染；
- 防止 CSRF、XSS、SSRF 和越权；
- 设备和 Token 可撤销。

### NFR-004 数据完整性

- 业务写入使用事务；
- 同步操作幂等；
- 重要实体有 version；
- 笔记有 CRDT 更新和可读快照；
- 数据迁移和恢复必须测试。

### NFR-005 可维护性

- 领域模块化；
- OpenAPI 契约；
- Alembic 迁移；
- 核心业务测试覆盖率不低于 85%；
- 重要决策使用 ADR；
- 功能开关支持逐步发布。

### NFR-006 可迁移性

- 核心文本使用 Markdown；
- 完整 JSON 导出；
- 指标和列表支持 CSV；
- 论文支持 BibTeX；
- 不把数据锁死在特定 AI 或前端框架。

### NFR-007 可访问性

- 前端实现至少满足 WCAG 2.1 AA 的主要要求；
- 键盘可操作；
- 状态不能只依赖颜色；
- 移动端支持核心流程。

## 6. 第一版发布范围

第一版包含：

1. 账户、Passkey、TOTP、恢复码和设备；
2. PWA、本地数据库、离线解锁和同步；
3. 学习计划、阶段、知识点和任务；
4. 导入 47 天计划；
5. 学习会话、笔记、资料、附件、证据和验收；
6. 掌握度、复习、测验和审查；
7. 论文和实验基础登记；
8. AI Provider、模型、路由和草稿；
9. 导入、导出、备份和审计。

第一版不包含：Git/Vigils 自动集成、公共分享、导师账户、视频托管、完整 PDF 托管和原生移动应用。

## 7. 发布验收总场景

1. 用户用 Passkey 登录电脑并导入 47 天计划；
2. 手机在线登录并完成首次同步；
3. 两台设备断网；
